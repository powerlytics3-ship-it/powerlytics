# 08 — API Specifications

Base URL: `https://api.powerlytic.app` (production), `http://localhost:4000` (local). All human-facing routes are versioned under `/v1`. Device-facing routes are versioned under `/v1/device`. The full machine-readable contract is generated by `@nestjs/swagger` from the controllers described here and published at `/v1/openapi.json`; both frontend apps generate a typed client from that spec (`packages/api-client`, regenerated on every backend change — see `19-coding-standards.md` for the contract-first workflow this replaces the current "no shared types" situation with).

Every table below states the **permission** required (from `07-authorization-rbac-design.md`'s matrix) rather than a role list directly — roles map to permissions, not the other way around.

## 11. Standard Response Shapes

```ts
// Success — list
interface PaginatedResponse<T> {
  data: T[];
  total: number;
  page: number;
  pageSize: number;
}

// Error — uniform across the API (replaces the current system's inconsistent
// {error} / {message} / {error, message} shapes used in different controllers)
interface ApiError {
  statusCode: number;
  code: string;        // machine-readable, e.g. "DEVICE_MODEL_PUBLISHED_IMMUTABLE"
  message: string;     // human-readable
  details?: Record<string, unknown>;
}
```

Every controller throws a small set of typed exceptions (`ForbiddenException`, `NotFoundException`, `ConflictException`, `BadRequestException` from Nest, plus a few domain-specific subclasses like `ImmutableResourceException`), caught by one global exception filter that maps them to the `ApiError` shape above. This replaces the current ad hoc `res.status(400/500).json({...})` calls scattered through every controller.

## 12. Request Validation

Every `POST`/`PATCH` body is a class decorated with `class-validator` + `class-transformer`, wired through Nest's global `ValidationPipe`. Example:

```ts
export class CreateAlertRuleDto {
  @IsString() deviceId: string;
  @IsString() portKey: string;
  @IsString() @MaxLength(120) name: string;
  @IsEnum(AlertCondition) condition: AlertCondition;
  @IsNumber() thresholdValue: number;
  @IsInt() @Min(0) forDurationSec: number;
  @IsEnum(AlertSeverity) severity: AlertSeverity;
}
```

This closes the "no request body validation library anywhere" gap noted in the existing-system analysis — malformed input now fails fast with a clear 400 and field-level error detail, instead of surfacing as an opaque 500 from a database-level cast error.
